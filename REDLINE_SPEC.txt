Final Architecture (Personal Tool — Windows)
Windows Task Scheduler (runs daily)
         ↓
redline/pipeline.py
         ↓
1. Poll SEC EDGAR for new filings (your watchlist tickers)
         ↓
2. Extract sections via sec-api.io (1A, 7, 3, 9A)
         ↓
3. Structural diff → JSON
         ↓
4. Pattern signal detection → appended to JSON
         ↓
5. GPT-OSS 20B via Ollama → reads JSON → outputs analysis
         ↓
6. Store everything in SQLite (redline.db)
         ↓
Simple local Flask webpage (localhost:5000)
reads from redline.db → displays results
No cloud. No auth. No Stripe. No Redis. No Celery. No PostgreSQL.
Just Python + SQLite + Ollama + a minimal Flask UI. Entire thing runs on your Windows machine.

Agent Task Specifications
These are written so Opus 4.6 / Codex can execute them one at a time with zero ambiguity. Each task has a single responsibility, clear inputs, clear outputs, and explicit acceptance criteria.

CREATE TABLE IF NOT EXISTS companies (
    cik         TEXT PRIMARY KEY,
    ticker      TEXT UNIQUE NOT NULL,
    name        TEXT NOT NULL,
    added_at    TEXT NOT NULL   -- ISO datetime
);

CREATE TABLE IF NOT EXISTS filings (
    id                TEXT PRIMARY KEY,  -- accession number, dashes removed
    cik               TEXT NOT NULL,
    ticker            TEXT NOT NULL,
    form_type         TEXT NOT NULL,     -- '10-K' or '10-Q'
    period_of_report  TEXT NOT NULL,     -- 'YYYY-MM-DD'
    filed_at          TEXT NOT NULL,     -- ISO datetime
    filing_url        TEXT NOT NULL,
    processed         INTEGER DEFAULT 0, -- 0 = not yet, 1 = done
    created_at        TEXT NOT NULL,
    FOREIGN KEY (cik) REFERENCES companies(cik)
);

CREATE TABLE IF NOT EXISTS sections (
    id          TEXT PRIMARY KEY,   -- uuid4
    filing_id   TEXT NOT NULL,
    section     TEXT NOT NULL,      -- '1A', '7', '3', '9A'
    raw_text    TEXT NOT NULL,
    word_count  INTEGER NOT NULL,
    created_at  TEXT NOT NULL,
    FOREIGN KEY (filing_id) REFERENCES filings(id),
    UNIQUE(filing_id, section)
);

CREATE TABLE IF NOT EXISTS diffs (
    id                      TEXT PRIMARY KEY,   -- uuid4
    current_filing_id       TEXT NOT NULL,
    previous_filing_id      TEXT NOT NULL,

    pct_changed             REAL NOT NULL,
    word_count_delta        INTEGER NOT NULL,

    sentences_removed       INTEGER NOT NULL,

    FOREIGN KEY (current_filing_id) REFERENCES filings(id),

TASK 005 — Structural Differ
Agent: Opus 4.6
Depends on: TASK 001
File: redline/differ.py
Description: Pure Python. No external APIs. Compare two text strings at the sentence level and produce a structured diff result.
Output dataclass:
@dataclass
class DiffResult:
    pct_changed: float          # 0.0 to 1.0
    word_count_old: int
    word_count_new: int
    word_count_delta: int       # positive = grew, negative = shrunk
    sentences_added: int
    sentences_removed: int
    sentences_unchanged: int
    diff_preview: str           # compact string, max 3000 chars
                                # format: lines prefixed with + or -
                                # only changed sentences, not unchanged
    raw_chunks: list[dict]      # full diff for storage
                                # each: {type, text} where type is
                                # 'added'|'removed'|'unchanged'
Algorithm requirements:

Split text into sentences using period/newline boundaries
Use difflib.SequenceMatcher at sentence level
diff_preview includes only added/removed chunks, truncated to 3000 chars with [...truncated] marker if longer
pct_changed = (sentences_added + sentences_removed) / max(total_old, total_new)

Acceptance criteria:

Identical texts return pct_changed=0.0, zero sentences added/removed
Completely different texts return pct_changed=1.0
diff_preview never exceeds 3000 chars
100% unit test coverage for the DiffResult dataclass
Test cases must include: identical, minor change, major change, one empty string

TASK 007 — Preliminary Scorer
Agent: Codex
Depends on: TASK 005, TASK 006
File: redline/scorer.py
Description: Single function that produces a score 1–10 before the LLM runs. Gates whether LLM gets called at all.
def preliminary_score(diff: DiffResult, signals: SignalResult) -> int:
    """
    Scoring logic:
    
    Base from structural diff (max 4 points):
      pct_changed > 0.30 → +4
      pct_changed > 0.15 → +3
      pct_changed > 0.05 → +2
      pct_changed > 0.01 → +1
      else               → +0
    
    From signals (max 5 points):
      max_severity >= 9  → +5
      max_severity >= 7  → +4
      max_severity >= 5  → +3
      max_severity >= 3  → +2
      triggered = True   → +1
      else               → +0
    
    Minimum: 1. Maximum: 10. No float — return int.
    """
Acceptance criteria:

Score is always between 1 and 10 inclusive
Going concern signal always produces score >= 7
Zero change + no signals always produces score = 1
10 unit tests covering edge cases

TASK 009 — Master Pipeline
Agent: Opus 4.6
Depends on: All previous tasks
File: redline/pipeline.py
Description: Orchestrates the full pipeline end to end. This is the entry point for the cron job.
Full flow:
def run():
    """
    1. Load watchlist.json
    2. init_db()
    3. For each ticker in watchlist:
       a. get_cik(ticker)
       b. upsert_company(cik, ticker, name)
       c. get_new_filings(ticker, form_types)
       d. For each new filing:
          i.   insert_filing(filing)
          ii.  extract_all_sections(filing_url, form_type, sections)
          iii. For each extracted section:
               - insert_section(filing_id, section, text)
               - get_section(previous_filing_id, section)
               - if previous section exists:
                   * diff = differ.diff(old_text, new_text)
                   * signals = signals.detect(old_text, new_text)
                   * prelim = scorer.preliminary_score(diff, signals)
                   * if prelim >= LLM_THRESHOLD:
                       llm_output = analyzer.analyze(...)
                   * else:
                       llm_output = None
                   * final_score = compute_final_score(prelim, llm_output)
                   * insert_diff(diff_dict)
                   * if llm_output: update_diff_llm(diff_id, final_score, llm_output)
               - else:
                   log("No previous filing — stored as baseline")
          iv. mark filing as processed
    4. Log summary: X filings processed, Y diffs created, Z LLM analyses run
"""

if __name__ == "__main__":
    run()
Acceptance criteria:

python -m redline.pipeline runs end to end with no unhandled exceptions
Idempotent — running twice on same day produces no duplicate records
Full logging at each step with timestamps
Summary printed at end
